# A13 — The dashboard leads with one Next Task

**Date:** 2026-05-25
**Status:** Accepted
**Rule:** R13

## Context

Pre-Copilot `/v2/home` was a grid of feature tiles. Users had to decide what to do; many left without acting.

## Decision

Sprint 6 replaces the home grid with a single "Next Task" card occupying the top of the page. Below the fold (scroll required) live secondary surfaces: recent activity, draft library, wallet summary.

The Next Task card text is generated nightly by the `next-task` prompt running on `claude-haiku-4-5-20251001` over the user's last-30-days activity log. The output is a single suggested action, written in the Wassel voice, with a primary CTA.

## Consequences

- `next_task_cards` Supabase table stores one row per user per day
- The Sprint 6 cron job runs nightly at 02:00 AST
- The Dashboard page (`/v2/home`) becomes a thin renderer

## Alternatives Rejected

- Five tile dashboard — rejected, original problem
- A/B test the layout — rejected, decision is product-level, not optimization

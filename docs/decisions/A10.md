# A10 — The Resume is targeted, not generic

**Date:** 2026-05-25
**Status:** Accepted
**Rule:** R10

## Context

A generic "build my CV" workflow produces a CV that is neither ATS-optimized for any specific role nor differentiated from any other Wassel user's.

## Decision

The Resume builder requires a `target_role` before it generates. It reads from:
1. `career_profile.target_role` (default)
2. An active `section_overrides` row with `section='resume'` (if present)

If neither exists, the UI prompts the user to declare a target role for the resume specifically (and offers to save it back to `career_profile` or keep it as a one-off override).

The Sonnet prompt for resume generation includes the target role, the user's experience, and a job-description-style brief derived from `career_profile`.

## Consequences

- `cv.generate` router signature changes to require `target_role` (Sprint 4)
- The Resume UI shows a chip indicating the active target role and a "تغيير الدور" link
- ATS keyword extraction in Sprint 4 keys off the target role

## Alternatives Rejected

- Allow generic CV generation — rejected, dilutes the differentiator
- Generate three CVs in parallel for three roles — rejected, costly and confusing

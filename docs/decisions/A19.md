# A19 — Bilingual at every commit

**Date:** 2026-05-25
**Status:** Accepted
**Rule:** R19

## Context

"Translate it later" is a tar pit. Once a feature ships with one language only, the second language never catches up.

## Decision

Every UI string requires both AR and EN keys in `translation.json` at the moment the PR is opened. A PR that adds keys only in `ar/translation.json` (or vice versa) is incomplete.

The CI check that enforces this lives in Sprint 8 (it requires the linter scaffolding). Until then, the rule is enforced by code review.

Career Copilot's namespaces are listed in `docs/language-rules.md` §10. AI Workforce session owns its own namespaces, listed in `docs/decisions/A22.md`.

## Consequences

- `client/public/locales/ar/translation.json` and `…/en/translation.json` ship together
- The build-prompts script (Sprint 1) does not gate on translations, but Sprint 8 CI will

## Alternatives Rejected

- Auto-translate via the AI — rejected, output quality is uneven for product copy
- Ship AR-first, add EN later — rejected, original problem

# A01 — Wassel is a Career Copilot, not a portfolio of tools

**Date:** 2026-05-25
**Status:** Accepted
**Rule:** R01

## Context

Pre-Copilot Wassel was perceived by users as four separate AI tools (profile analyzer, CV builder, post generator, dashboard). Each had its own onboarding, its own data input, and no awareness of the others.

## Decision

Wassel reframes itself as a single product: a Career Copilot that leads the user through a single career journey. The four pillars (Radar, Resume, Content, Dashboard) are *aspects* of that journey, not standalone tools. Every screen is responsible for advancing the user toward their declared career goal.

## Consequences

- Onboarding (Sprint 2) writes a `career_profile` row that every other screen reads
- The Dashboard becomes a "Next Task" card, not a feature menu (R13)
- Marketing copy stops advertising features as separable products
- The pricing page describes journeys, not feature licenses

## Alternatives Rejected

- "Keep selling four tools, add cross-links between them" — rejected because users still feel siloed
- "Bundle the four tools under a single brand but keep separate flows" — rejected for the same reason

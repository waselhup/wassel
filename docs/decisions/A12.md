# A12 — First 5 refinements per document are free

**Date:** 2026-05-25
**Status:** Accepted
**Rule:** R12

## Context

Resume and Content outputs are first drafts. Asking the user to spend tokens to refine paragraphs they've already paid to generate feels punitive.

## Decision

The first **5 refinement actions per document** (e.g. "refine this paragraph", "shorten the summary", "rewrite in a more confident tone") cost zero tokens. From the 6th refinement onward, each refinement costs a small published rate (TBD in Sprint 7, target ~5 tokens).

Refinement count is tracked per document in `document_refinement_count`, not per user. So a user with five documents has 25 free refinements available.

## Consequences

- A counter table or column tracks per-document refinements
- The refinement UI shows "4 من 5 تعديلات مشمولة" / "4 of 5 refinements included"
- Sprint 7 finalizes the post-5 rate

## Alternatives Rejected

- All refinements free — rejected, abusable for low-effort iteration
- Pay per refinement from the start — rejected, punishes legitimate iteration

# A21 — Prompts are markdown, not code

**Date:** 2026-05-25
**Status:** Accepted
**Rule:** R21

## Context

Prompts inline in TypeScript routers produce:
- Unreadable diffs (escapes, template literals, indentation)
- Friction for product / copy edits (every change is a TS PR)
- No single place to look for "what does the model see?"

## Decision

Prompts live in `docs/prompts/*.md`. Each file has three fenced blocks:

````markdown
```system
<system prompt>
```

```user
<user-message template with {{placeholders}}>
```

```schema
<TypeScript-flavored output schema>
```
````

`scripts/build-prompts.ts` parses each file and emits a TypeScript object into `server/_core/prompts/_generated.ts`:

```ts
export const radarPrompt = {
  system: "...",
  user: (vars) => `...`,
  schema: "...",
} as const;
```

Routers call `radarPrompt.system` and `radarPrompt.user({...})`. No router instantiates a raw string for a model call.

## Consequences

- Editing a prompt is editing a markdown file
- The build script must run before `npm run build` (wired in `package.json`)
- The generated file is regenerable; do not hand-edit it

## Alternatives Rejected

- Prompts in `.json` — rejected, no comments, hard to write
- Prompts in `.yaml` — rejected, whitespace fragility
- Prompts as TypeScript template literals — original problem

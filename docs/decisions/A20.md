# A20 — The build is repeatable

**Date:** 2026-05-25
**Status:** Accepted
**Rule:** R20

## Context

`api/index.js` has been edited by hand in the past, leading to drift between source (`server/_core/`) and shipped bundle. Each manual edit was a small mystery for the next engineer.

## Decision

`api/index.js` is **always** the output of:

```
npx esbuild server/_core/vercel.ts \
  --platform=node --bundle --format=cjs \
  --outfile=api/index.js \
  --external:@napi-rs/canvas* \
  --external:pdfjs-dist
```

A PostToolUse hook (`.claude/settings.local.json`) already auto-rebuilds on `server/_core/*` changes. No manual edits are allowed.

`scripts/build-prompts.ts` is the second piece of the repeatable build: it regenerates `server/_core/prompts/_generated.ts` from `docs/prompts/*.md`. The `build:prompts` script is wired into `package.json` (Sprint 1).

A clean clone of the repo + `npm install` + `npm run build:prompts` + esbuild should produce a working deployable.

## Consequences

- `api/index.js` is regenerated on every relevant change
- The esbuild externals list is documented in CLAUDE.md
- The prompt build is committed to git (the generated file lives in the repo for now; Sprint 8 may move it to a build artifact)

## Alternatives Rejected

- Track `api/index.js` as source — rejected, drift risk
- Skip the prompt build, write prompts in TS — rejected, contradicts R21

# A08 — Onboarding lands inside the Radar

**Date:** 2026-05-25
**Status:** Accepted
**Rule:** R08

## Context

A common SaaS antipattern: onboarding ends on a dashboard full of feature tiles. The new user has done work (told you their goal) and is rewarded with… a menu. They leave.

## Decision

The 4-step onboarding wizard ends with a "complete" screen whose single primary CTA is **"ابدأ تحليل الرادار الأول"** — clicking it routes directly to the Radar for the role the user just declared, pre-filled.

If the user skipped LinkedIn in step 4, the Radar shows the manual-input flow; otherwise it auto-runs the discovery and shows results.

## Consequences

- `OnboardingComplete.tsx` has a single primary CTA, not three
- Sprint 3 must support a "Radar pre-filled from onboarding" entry mode
- The Dashboard is never the post-onboarding destination

## Alternatives Rejected

- End on a dashboard with a banner — rejected, dilutes the CTA
- Let the user choose Radar / Resume / Content — rejected, violates R01 (lead, don't offer)

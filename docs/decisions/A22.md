# A22 — Parallel-session fences are hard, not soft

**Date:** 2026-05-25
**Status:** Accepted
**Rule:** R22

## Context

Two Claude Code sessions are working in parallel on Wassel:

1. **Career Copilot session** (this branch: `feat/career-copilot-transformation`) — builds onboarding, career_profile, 3-wallet system, docs, settings.
2. **AI Workforce session** (a separate branch) — builds Sayed, Faris, Al-Mukhadram, Hassan, Fatima, Dhai, Hussein, Mohammed agents + Customer Success / Revenue Lab / Product Intel / Compliance portals.

Without hard fences, the two sessions trample each other's files and cause merge nightmares.

## Decision

Each session has a written blocklist. This decision documents the **Career Copilot blocklist** — files this session must NOT touch.

### What to do if a conflict is unavoidable

1. **Stop.** Do not push a workaround.
2. **Read** the AI Workforce session's status (visible in git log on its branch when merged).
3. **Ask Ali** — `STOP&ASK` event in CLAUDE.md terms.
4. **Coordinate** an explicit resolution. Do not "guess and hope".

## Consequences

- Career Copilot's git history stays within the listed files
- AI Workforce's git history stays within its own files
- Merges to master happen sequentially, never in parallel (Vercel cancels parallel deploys anyway — separate problem, same direction)

## Alternatives Rejected

- "Soft" advisory list — rejected, too easy to ignore
- One session at a time — rejected, doubles delivery time

# A07 — Western digits everywhere

**Date:** 2026-05-25
**Status:** Accepted
**Rule:** R07

## Context

Arabic UIs default to Arabic-Indic digits via `toLocaleString('ar-SA')`. The Wassel team and users tested both; the consensus from the user-research panel (n=18 in Riyadh + Jeddah, April 2026) was overwhelming preference for Western digits in professional career contexts.

## Decision

Western digits 0–9 everywhere, in both AR and EN UIs. This includes:
- Token counts
- Prices
- Phone numbers
- Dates
- Times
- Percentages
- Statistics in Radar / Resume output

A `formatNumber(n, locale)` helper in `client/src/lib/format.ts` always returns Western digits regardless of locale.

## Consequences

- Existing `toLocaleString` calls are audited and replaced
- AI prompts include the instruction "Always use Western digits (0-9), not Arabic-Indic (٠-٩)"
- Export PDFs use Western digits

## Alternatives Rejected

- Honor `toLocaleString('ar-SA')` — rejected by user research
- Make it a Settings toggle — rejected as feature bloat for a clear preference
